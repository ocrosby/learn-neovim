# Module 5: Language Server Protocol (LSP)

Transform Neovim into a full-fledged IDE with LSP support for code completion, diagnostics, go-to-definition, and more.

## Learning Objectives

- Understand what LSP is and why it matters
- Set up nvim-lspconfig
- Install language servers with Mason
- Configure LSP features (completion, diagnostics, etc.)
- Use LSP keybindings effectively

## Estimated Time

2-4 hours

## Prerequisites

- Completed [Module 4: Plugin Management](../04-plugin-management/README.md)
- Comfortable with plugin installation
- Have lazy.nvim set up

## Lessons

1. **[What is LSP?](what-is-lsp.md)**
2. **[Installing Language Servers with Mason](mason.md)**
3. **[Configuring LSP](configuration.md)**

## Why LSP?

LSP gives you IDE features:
- Autocompletion
- Go to definition
- Find references
- Diagnostics (errors/warnings)
- Code actions (quick fixes)
- Hover documentation

All language-agnostic and standardized.
